# 快速开始 - 可视化远程控制

## 一分钟上手

### 步骤1：启动被控端（服务端）

```bash
python server_gui.py
```

在弹出的窗口中：
1. 点击 **"START SERVICE"** 按钮
2. 记下显示的本机IP地址（如：192.168.1.100）

![Server](https://via.placeholder.com/400x300/1e1e1e/00ff00?text=Server+GUI)

---

### 步骤2：启动控制端（客户端）

#### 方式A：使用快速启动脚本
```bash
python start_visual_client.py
```

#### 方式B：直接运行
```bash
python client_visual.py
```

---

### 步骤3：连接到目标

在客户端登录界面：
1. 输入目标IP（被控端的IP）
2. 输入密码（默认：`Yssx`）
3. 点击 **"ESTABLISH CONNECTION"**

![Login](https://via.placeholder.com/400x300/1e1e1e/00ff00?text=Login+Screen)

---

### 步骤4：授权连接

被控端会弹出授权窗口：
- 显示请求来源的IP和端口
- 点击 **"是"** 允许连接
- 点击 **"否"** 拒绝连接

---

### 步骤5：开始使用可视化功能

连接成功后，你将看到6个功能标签页：

#### 📊 仪表盘（自动运行）
- **无需操作**，自动显示：
  - CPU使用率
  - 内存使用率
  - 磁盘使用率
  - 网络连接数
  - 进程数量
  - 系统运行时间

#### ⚙️ 进程管理（点击即用）
1. 点击 **"REFRESH"** 获取进程列表
2. 选择要结束的进程
3. 点击 **"KILL PROCESS"**

#### 📁 文件浏览（点击即用）
1. 在路径框输入目录（如：`/home/user`）
2. 点击 **"GO"**
3. 双击文件夹浏览
4. 选择文件后点击：
   - **Download** 下载
   - **View** 查看
   - **Delete** 删除

#### 🖥️ 屏幕监控（点击即用）
- 点击 **"START STREAM"** 开始实时监控
- 点击 **"SCREENSHOT"** 截取单帧
- 点击 **"STOP STREAM"** 停止

#### 💻 Shell终端（点击即用）
- 直接输入命令，按回车执行
- 例如：`ls -la`, `pwd`, `ps aux`

#### 📜 操作历史（自动记录）
- 所有操作自动记录
- 点击 **"EXPORT"** 导出日志

## 典型使用场景

### 场景1：监控系统资源
1. 保持在 **"📊 DASHBOARD"** 标签页
2. 观察CPU、内存、磁盘使用率
3. 进度条颜色：
   - 🟢 绿色 = 正常（<60%）
   - 🟠 橙色 = 警告（60-80%）
   - 🔴 红色 = 危险（>80%）

### 场景2：结束卡顿进程
1. 切换到 **"⚙️ PROCESSES"** 标签页
2. 点击 **"REFRESH"** 刷新列表
3. 查看CPU%和Memory%列，找到占用高的进程
4. 选中后点击 **"KILL PROCESS"**

### 场景3：下载远程文件
1. 切换到 **"📁 FILES"** 标签页
2. 浏览到文件所在目录
3. 选中文件
4. 点击 **"Download"**
5. 选择保存位置

### 场景4：实时监控屏幕
1. 切换到 **"🖥️ SCREEN"** 标签页
2. 调整质量滑块（推荐50）
3. 点击 **"START STREAM"**
4. 实时画面显示在窗口中
5. 完成后点击 **"STOP STREAM"**

### 场景5：执行批量命令
1. 切换到 **"💻 SHELL"** 标签页
2. 依次输入命令：
   ```bash
   cd /tmp
   ls -la
   cat log.txt
   ```
3. 每个命令的输出都会显示

### 场景6：审计操作记录
1. 切换到 **"📜 HISTORY"** 标签页
2. 查看所有操作的时间、类型、详情
3. 点击 **"EXPORT"** 导出为文本文件
4. 用于审计或问题追溯

## 安全建议

1. **修改默认密码**
   编辑 `config.py`：
   ```python
   AUTH_PASSWORD = 'your_strong_password'
   ```

2. **限制访问IP**
   在服务端添加IP白名单

3. **使用VPN**
   在公网环境下建议通过VPN连接

4. **定期检查日志**
   在 **"📜 HISTORY"** 中查看所有操作记录

## 故障排除

### ❌ 连接失败
**可能原因**：
- IP地址错误
- 端口被防火墙阻止
- 密码错误

**解决方法**：
1. 确认被控端已启动服务
2. ping 目标IP确认网络连通
3. 检查防火墙设置（允许端口9999）
4. 验证密码是否正确

### ❌ 功能无响应
**可能原因**：
- 网络中断
- 服务端崩溃

**解决方法**：
1. 查看 **"📜 HISTORY"** 中的错误信息
2. 检查网络连接状态
3. 重新连接

### ❌ 屏幕流很卡
**解决方法**：
1. 降低质量滑块（调到30-40）
2. 检查网络带宽
3. 使用 **"SCREENSHOT"** 代替实时流
